Metáfora

La herencia es como el ADN

🧬

Un hijo hereda características de sus padres:
color de ojos, altura, tipo de sangre…

Pero también tiene sus propias características que lo hacen único.

🖥️

Una clase derivada hereda atributos y métodos de la clase base.

Puede extenderlos y especializarlos sin reescribir lo que ya existe.

💡 Principio clave: La herencia modela una relación "ES UN" — un Perro es un Animal, un Auto es un Vehículo, un Paquete es un Envío.
El problema

Sin herencia: código duplicado

❌ Sin herencia

class Perro {
  string nombre;
  int    edad;
  void   comer();
  void   dormir();
  void   ladrar();
};

class Gato {
  string nombre;  // duplicado
  int    edad;    // duplicado
  void   comer();  // duplicado
  void   dormir(); // duplicado
  void   maullar();
};

✅ Con herencia

class Animal {           // base
  string nombre;
  int    edad;
  void   comer();
  void   dormir();
};

class Perro : public Animal {
  void ladrar();   // solo lo nuevo
};

class Gato : public Animal {
  void maullar();  // solo lo nuevo
};
Sintaxis

Cómo se declara la herencia

// ┌── Clase derivada    ┌── Tipo de acceso   ┌── Clase base
class ClaseHija     :    public           ClaseBase
{
    // Nuevos atributos y métodos específicos de ClaseHija
};
public

Los miembros public y protected de la base mantienen su acceso. El más común.

protected

Los miembros public de la base se vuelven protected en la derivada.

private

Todo lo heredado se vuelve private. Útil para implementación interna.

💡 Para la tarea y la mayoría de proyectos reales: usa siempre public

Constructores

Orden de ejecución y lista de inicialización

class Animal {
protected:
    string nombre;
    int    edad;
public:
    Animal(string n, int e)
        : nombre(n), edad(e) {}
};

class Perro : public Animal {
private:
    string raza;
public:
    Perro(string n, int e, string r)
        : Animal(n, e),  // ① llama al padre
          raza(r) {}      // ② inicializa lo propio
};

Orden garantizado

  1. Constructor de la clase base
  2. Constructor de la clase derivada
  3. Destructores en orden inverso
💡 Usa : Base(args) en la lista de inicialización — es la única forma de pasar parámetros al constructor del padre.
Tipos

Tipos de herencia en C++

A → B
Simple

Una derivada hereda de una sola base. El más común y sencillo.

A → C ← B
Múltiple

Una clase hereda de dos o más bases. Cuidado con el diamante.

A → B → C
Multinivel

Cadena de herencia: nieto hereda de hijo que hereda de abuelo.

A → B, A → C
Jerárquica

Múltiples clases derivan de una sola base. Usada en la tarea.

A → B ← C ← A
Híbrida

Mezcla de tipos anteriores. Usa virtual para evitar ambigüedad.

Sobreescritura

Redefinir comportamiento

class Envio {
public:
    virtual double calculaCosto() {
        return costoEstandar;
    }
};

class Paquete : public Envio {
public:
    double calculaCosto() override {
        // peso * costoPorKg + costoBase
        return Envio::calculaCosto()
             + peso * costoPorKg;
    }
};

Palabras clave importantes

virtualEn la base: habilita polimorfismo en runtime
overrideEn la derivada: el compilador verifica que sí existe en la base
Base::fn()Llama a la versión del padre dentro del hijo
💡 override es tu mejor amigo: si cambias el nombre en la base, el compilador te avisa en lugar de crear silenciosamente una función nueva.
Industria Real

¿Dónde se usa herencia en producción?

🎮 Motores de videojuegos

Unreal Engine / Unity: Actor → Pawn → Character → MyPlayer. Todo personaje hereda física, colisión e input del engine y solo especializa su comportamiento.

🖼️ Frameworks UI

Qt: QObject → QWidget → QPushButton. Un puntero QWidget* puede apuntar a cualquier componente visual y llamar render() polimórficamente.

☁️ Cloud / APIs REST

SDKs como AWS: BaseClient → S3Client, BaseClient → DynamoClient. Autenticación, retry y logging viven en la base; cada servicio solo implementa sus endpoints.

📦 Sistemas de envío

FedEx, UPS, DHL internamente modelan exactamente lo de la tarea: Shipment base con costo estándar, derivadas Envelope, Package, Freight con reglas propias.

Código — Clase Base

Envio.h — La clase base

// Envio.h
#pragma once
#include <string>
using namespace std;

class Envio {
protected:
    // Datos del remitente
    string nomRem, dirRem, ciudRem, estRem, cpRem;
    // Datos del destinatario
    string nomDest, dirDest, ciudDest, estDest, cpDest;
    double costoEstandar;

public:
    Envio(string nR, string dR, string cR, string eR, string pR,
          string nD, string dD, string cD, string eD, string pD,
          double costo)
        : nomRem(nR), dirRem(dR), ciudRem(cR), estRem(eR), cpRem(pR),
          nomDest(nD), dirDest(dD), ciudDest(cD), estDest(eD), cpDest(pD),
          costoEstandar(costo) {}

    virtual double calculaCosto() const {
        return costoEstandar;
    }

    virtual ~Envio() = default;
};
Código — Clase Derivada

Paquete.h — Hereda de Envio

// Paquete.h
#pragma once
#include "Envio.h"

class Paquete : public Envio {
private:
    double largo, ancho, profundidad;
    double peso;
    double costoPorKg;

public:
    Paquete(string nR, string dR, string cR, string eR, string pR,
             string nD, string dD, string cD, string eD, string pD,
             double costo,
             double l, double a, double prof,
             double p, double cpk)
        : Envio(nR,dR,cR,eR,pR, nD,dD,cD,eD,pD, costo),  // ← llama al padre
          largo(l), ancho(a), profundidad(prof),
          peso(p > 0 ? p : 0),
          costoPorKg(cpk > 0 ? cpk : 0) {}

    double calculaCosto() const override {
        return Envio::calculaCosto() + peso * costoPorKg;
    }
};
⚠️ La tarea pide validar que peso y costoPorKg sean positivos — el operador ternario lo hace en una línea.
Código — Clase Derivada

Sobre.h — Hereda de Envio

// Sobre.h
#pragma once
#include "Envio.h"

class Sobre : public Envio {
private:
    double largo;           // en centímetros
    double ancho;           // en centímetros
    double cargoAdicional;  // se aplica si supera 25×30 cm

public:
    Sobre(string nR, string dR, string cR, string eR, string pR,
           string nD, string dD, string cD, string eD, string pD,
           double costo,
           double l, double a, double cargo)
        : Envio(nR,dR,cR,eR,pR, nD,dD,cD,eD,pD, costo),
          largo(l), ancho(a), cargoAdicional(cargo) {}

    double calculaCosto() const override {
        double total = Envio::calculaCosto();
        // Cargo extra si cualquiera de las dimensiones supera el límite
        if (largo > 25.0 || ancho > 30.0)
            total += cargoAdicional;
        return total;
    }
};
Código — Prueba

main.cpp — Programa de prueba

#include <iostream>
#include <iomanip>
#include "Envio.h"
#include "Paquete.h"
#include "Sobre.h"
using namespace std;

int main() {
    cout << fixed << setprecision(2);

    // Envío base — solo costo estándar
    Envio envio("Ana Gómez","Av.Tec 100","Monterrey","NL","64000",
                "Luis Mora","Calle 5","CDMX","CDMX","06600",  50.0);
    cout << "Envio base:  $" << envio.calculaCosto() << "\n";

    // Paquete: 2 kg × $30/kg + $50 base = $110
    Paquete paq("Ana Gómez","Av.Tec 100","Monterrey","NL","64000",
                "Luis Mora","Calle 5","CDMX","CDMX","06600",  50.0,
                30.0, 20.0, 15.0,  // largo, ancho, prof
                2.0, 30.0);          // peso, costoPorKg
    cout << "Paquete:     $" << paq.calculaCosto() << "\n";

    // Sobre grande: dimensiones 30×35 → cargo adicional $20
    Sobre sobre("Ana Gómez","Av.Tec 100","Monterrey","NL","64000",
                "Luis Mora","Calle 5","CDMX","CDMX","06600",  50.0,
                30.0, 35.0, 20.0);  // largo, ancho, cargo
    cout << "Sobre grande: $" << sobre.calculaCosto() << "\n";
}
Bonus — Polimorfismo

Potencia real: un arreglo de Envios

#include <vector>
#include <memory>

// vector de punteros a la clase BASE
vector<unique_ptr<Envio>> envios;

envios.push_back(make_unique<Envio>(…));
envios.push_back(make_unique<Paquete>(…));
envios.push_back(make_unique<Sobre>(…));

// El mismo loop llama la versión CORRECTA de calculaCosto()
for (const auto& e : envios)
    cout << "Costo: $" << e->calculaCosto() << "\n";
// ↑ Envio::calculaCosto(), Paquete::calculaCosto(), Sobre::calculaCosto()
//   se resuelve en TIEMPO DE EJECUCIÓN gracias a `virtual`
🚀 Así funciona internamente el SDK de FedEx/UPS: un mismo bucle procesa miles de envíos de distinto tipo sin un solo if/switch.
Errores comunes

Trampas que debes evitar

❌ Olvidar llamar al constructor base

Paquete::Paquete(…)
    // sin : Envio(…) → costoEstandar queda basura
    : peso(p) {}

❌ Sin destructor virtual en la base

// Si Envio no tiene ~Envio() virtual
// y usas punteros base → memory leak
Envio* e = new Paquete(…);
delete e; // ← destructor incorrecto

⚠️ Olvidar override

// Sin override: si cambias la firma
// en la base, el compilador NO avisa
double calculaCosto() { … }  // peligroso
double calculaCosto() override { … } // seguro

⚠️ Atributos privados no accesibles

// Si costoEstandar es private
// Paquete no puede leerlo
// → usa protected o getter
Checklist

Lista de verificación antes de entregar

Clase Envio

  • ☐ Datos remitente (nombre, dir, ciudad, estado, CP)
  • ☐ Datos destinatario (igual)
  • ☐ costoEstandar como miembro protected
  • ☐ Constructor inicializa todo
  • ☐ virtual double calculaCosto()
  • ☐ Destructor virtual

Clase Paquete

  • ☐ Hereda de Envio con public
  • ☐ largo, ancho, profundidad, peso, costoPorKg
  • ☐ Validación peso > 0 y costoPorKg > 0
  • ☐ calculaCosto() override = base + peso×$/kg

Clase Sobre

  • ☐ Hereda de Envio con public
  • ☐ largo, ancho, cargoAdicional

Resumen

Lo que aprendiste hoy

♻️

Reutilización

Escribe código una vez en la base; todos los hijos lo heredan.

🔧

Especialización

Cada clase hija sobreescribe solo lo que necesita cambiar.

🔀

Polimorfismo

Un puntero base llama el método correcto en runtime.

🏆 La regla de oro: si puedes decir "X es un Y" en lenguaje natural, entonces tiene sentido que X herede de Y en código. Un Paquete es un Envio ✔
